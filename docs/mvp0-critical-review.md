# MVP0 비판적 검토 대화

## 범위

MVP0은 친구 기능, 서버 저장, 돌봄 시스템을 구현하지 않는다. 사용자가 혼자 포모도로를 기록하고, 오늘 집중 시간/횟수/포인트가 남으며, 기본 캐릭터가 반응하는지만 검증한다.

## 대화

**사용자:** 지금 화면에서 내가 뭘 해야 하는지는 보이지만, 실제 타이머가 카운트다운하지 않고 "완료로 기록" 버튼만 누르면 기록됩니다. 이러면 포모도로 앱이라고 하기 어렵지 않나요?

**앱 개발자:** 맞습니다. 현재는 기록 루프와 저장 검증을 먼저 보기 위한 MVP0입니다. 하지만 다음 개선에서는 "시작", "일시정지", "완료" 흐름을 넣어야 합니다. 지금 상태는 사용자 테스트용이라기보다 내부 클릭 검증용에 가깝습니다.

**사용자:** 그러면 MVP0의 성공 기준을 "사용자가 실제 포모도로를 쓴다"로 두면 안 됩니다. "사용자가 앱의 목적을 이해하고 기록이 저장되는지 확인한다"가 더 정확합니다.

**앱 개발자:** 동의합니다. 현재 성공 기준은 화면 이해도와 localStorage 기록 안정성입니다. 실제 집중 행동 검증은 타이머가 들어간 MVP0.5 또는 MVP1에서 해야 합니다.

**사용자:** 완료 버튼만 있으면 포인트를 무한히 올릴 수 있습니다. 포인트를 벌기 위한 앱처럼 보이면 나중에 게임 중독 방지 철학과 충돌하지 않나요?

**앱 개발자:** 지금 포인트는 보상 피드백만 보여주는 임시 값입니다. 다음 단계에서 실제 타이머 완료 조건, 하루 통계, 중복 방지 기준을 넣어야 합니다. 포인트 사용처도 아직 만들면 안 됩니다.

**사용자:** 캐릭터가 귀엽긴 한데, "집중하면 캐릭터가 좋아진다"는 연결이 약합니다. 그냥 반응 문구 하나뿐입니다.

**앱 개발자:** MVP0에서는 캐릭터 반응을 최소화했습니다. MVP3 최소 돌봄 단계에서 행복도, 배고픔, 표정 변화를 연결해야 합니다. 다만 지금도 완료 후 캐릭터 반응 문구가 바뀌므로 기본 방향은 확인할 수 있습니다.

**사용자:** 친구/캐릭터/스토어 탭이 열리지만 기능이 없다고 말합니다. 이게 실망감을 줄 수도 있지 않나요?

**앱 개발자:** 베타 사용자에게 그대로 내보내면 실망감을 줄 수 있습니다. 현재는 개발 순서를 검토하기 위한 앱 쉘입니다. 공개 테스트 전에는 잠긴 탭을 숨기거나 "곧 열림" 상태를 더 명확히 해야 합니다.

**사용자:** 디자인은 앱처럼 보이지만 모바일에서 실제로 오래 쓰기 편한지는 아직 모르겠습니다. 캐릭터가 큰 건 좋지만 타이머 조작이 하단에 고정되어 있지 않습니다.

**앱 개발자:** 다음 UI 검증에서 모바일 한 손 사용성을 봐야 합니다. 특히 시작/완료 버튼, 시간 선택, 오늘 통계를 홈 화면에서 스크롤 없이 볼 수 있는지가 중요합니다.

**사용자:** MVP0 다음에 바로 친구 경쟁으로 가도 되나요?

**앱 개발자:** 아직은 이릅니다. 실제 카운트다운 타이머가 없고, 세션 완료 신뢰성이 없습니다. 먼저 MVP0.5로 실제 타이머와 기록 검증을 끝낸 다음, MVP1 계정 저장으로 넘어가는 게 맞습니다.

## 개선 우선순위

1. 실제 카운트다운 타이머를 추가한다.
2. 완료 버튼은 타이머가 끝난 뒤에만 주요 행동으로 만든다.
3. 오늘 기록 초기화 기준을 명확히 한다.
4. 포인트 무한 획득 방지 규칙을 넣는다.
5. 모바일에서 타이머 조작 영역을 더 우선 배치한다.
6. 잠긴 탭은 베타 전까지 숨길지, 안내 상태로 둘지 결정한다.

## 결론

MVP0은 제품의 최종 재미를 검증하지 않는다. 현재 검증한 것은 "뽀모치가 어떤 앱인지 첫 화면에서 이해되는가", "혼자 집중 기록이 남는가", "기록에 캐릭터 반응과 포인트가 연결되는가"다. 다음 단계는 친구 기능이 아니라 실제 타이머 신뢰성이다.
