## 1. 제품 정의

### 한 줄 정의
> 친구와 약속하고 경쟁하며 집중 시간을 쌓고, 그 기록으로 캐릭터 유대를 만드는 소셜 포모도로 앱

### 핵심 가치 4가지

```
1. 함께 집중
   혼자 하면 쉽게 포기하는 포모도로를
   친구와 약속하고 같이 하면 완료율이 높아진다

2. 친구 경쟁
   친구와 오늘/이번 주 집중 시간을 비교하며
   가볍게 이기고 싶은 마음으로 다시 시작하게 만든다

3. 함께 키우기
   내 캐릭터 + 친구 캐릭터가 친밀도를 쌓고
   함께 노력한 결과로 새끼를 낳는다

4. 오래 정드는 관계
   빠르게 죽고 순환하는 것이 아니라
   1~2개월 천천히 키우며 진짜 애착이 생긴다
```

### 제품 철학

**양육은 배경, 소셜이 주인공.**
캐릭터를 키우는 재미는 있지만, 뽀모치의 진짜 재미는 친구와 함께 집중하고 그 결과로 만들어지는 유대다. 캐릭터는 그 유대를 눈에 보이게 만드는 매개다.

**죽지 않는 타마고치.**
기존 타마고치는 며칠이면 끝나 정들기 전에 사라진다. 뽀모치의 캐릭터는 절대 죽지 않는다. 방치하면 시무룩해지지만 돌봐주면 회복된다. 은퇴 후에도 가족 계보에 영원히 남는다.

**실시간 없는 함께.** 
정확한 동시 접속보다 "혼자가 아니라는 느낌"이 중요하다. 약속 세션, 공동 목표, 친구 집중 시간 경쟁만으로 그 느낌을 충분히 줄 수 있다.

## 2. 시장 & 차별점

### 기존 앱과의 비교

| 구분 | Forest | Finch | Habitica | **뽀모치** |
|------|--------|-------|---------|-----------|
| 집중 타이머 | ✅ | ❌ | ❌ | ✅ |
| 캐릭터 양육 | ❌ | ✅ | △ | ✅ |
| 소셜 함께 집중 | ✅ | ❌ | △ | ✅ **핵심** |
| 친구 집중시간 경쟁 | △ | ❌ | △ | ✅ **초기 핵심** |
| 친구 캐릭터 상호작용 | ❌ | ❌ | ❌ | ✅ **유일** |
| 함께 낳은 새끼 | ❌ | ❌ | ❌ | ✅ **유일** |
| 가족 계보 | ❌ | ❌ | ❌ | ✅ **유일** |
| 캐릭터 불사 | ❌ | ❌ | ❌ | ✅ |

### 핵심 차별점

```
1. 친구 집중시간 경쟁
   Forest도 함께 집중/전체 경쟁 요소가 있지만,
   뽀모치는 가까운 친구끼리 오늘/이번 주 집중 시간을 직접 비교한다.
   "누가 더 많이 집중했나"가 친구 탭의 기본 재미가 된다.

2. 친구와 함께 만든 캐릭터
   내 불꽃이 + 친구의 물방울이 → 증기형 새끼 탄생
   → 이 새끼는 우리 둘의 포모도로 노력으로 만들어진 것
   → 누구도 혼자 가질 수 없음
   → 자연스러운 락인 + 깊은 유대
```

## 3. 사용자 & 핵심 시나리오

### 타겟 사용자

**주 타겟:** 혼자 집중이 잘 안 되는 16~35세
- 학생: 시험 기간, 혼자 도서관 가기 싫은 사람
- 직장인: 재택근무 중 집중력 유지가 어려운 사람
- 공통: 친구와 함께라면 더 잘하는 사람

### 핵심 시나리오 3가지

**시나리오 A: 일상 루틴**
```
아침: "○○가 오늘 저녁 9시 같이 집중하자고 해요" 알림
낮: 각자 솔로 포모도로 (공동 목표 기여)
저녁 9시: 약속 알림 → 둘이 함께 포모도로
완료: "약속 지킴! 친밀도↑ +20pt"
```

**시나리오 B: 공동 목표 달성**
```
주초: 친구와 "이번 주 함께 50회" 목표 설정
주중: 각자 자기 페이스로 포모도로 기여
주말: 목표 달성 → "🥚 공동 알 획득!"
다음 주: 공동 알 부화 → 둘 닮은 새끼 탄생
```

**시나리오 C: 세대교체**
```
1개월: 불꽃이 (내 캐릭터) 어덜트 도달
6주: 충분한 친밀도 쌓임
7주: 친구의 물방울이와 번식 → 공동 알
8주: 증기형 새끼 탄생 (희귀!)
"불꽃이의 자식이 생겼어. 물방울이도 닮았네"
```

## 4. 기능 명세

### 4.1 알 부화 & 캐릭터

#### 신규 가입 흐름
```
1. 신비한 알 지급
2. "부화하기" 버튼 → 랜덤 캐릭터 등장
3. 리롤 최대 3회 (원하는 캐릭터 선택권)
4. 이름 짓기 (애착 시작)
5. 온보딩 시작
```

#### 캐릭터 5종

| 이름 | 타입 | 실루엣 특징 | 색상 | 성격 |
|------|------|-----------|------|------|
| 불꽃이 | 🔥 불꽃형 | 뿔 + 꼬리 | 레드/오렌지 | 활발, 에너지 |
| 물방울이 | 💧 물방울형 | 물방울 실루엣 | 블루/민트 | 차분, 상냥 |
| 새싹이 | 🌿 자연형 | 토끼귀 + 새싹 | 그린 | 밝고 긍정적 |
| 번개 | ⚡ 번개형 | 각짐 + 번개 안테나 | 옐로우 | 장난기, 날쌤 |
| 달빛이 | 🌙 달빛형 | 뿔 + 날개 | 퍼플 | 신비, 조용함 |

**캐릭터 설계 원칙:**
- 화면의 60% 이상 차지 (한눈에 보임)
- 실루엣만 봐도 구별 가능
- 표정 3단계: 행복(happy) / 평상시(idle) / 아픔(sick)
- 상태에 따라 표정 자동 변화

#### 혼합 캐릭터 (번식 결과, Phase 2)

| 조합 | 결과 | 희귀도 |
|------|------|--------|
| 🔥 × 💧 | 🌫️ 증기형 | 희귀 |
| 🔥 × 🌿 | 🌋 용암형 | 희귀 |
| 💧 × 🌙 | 🌊 심해형 | 희귀 |
| ⚡ × 🌙 | 🌟 오로라형 | 초희귀 |
| 같은 타입 × 같은 타입 | 동일 타입 | 일반 |

### 4.2 생애주기

#### 단계 & 기간

```
🥚 알        1~2일    부화 대기
👶 베이비    3~7일    손이 많이 감 (애착 형성)
🧒 차일드    8~20일   성격 형성, 포모도로 패턴 반영
🧑 어덜트    21~40일  완전체, 번식 가능
👴 엘더      41~55일  차분, 마지막 번식 기회
🏡 은퇴      55일+    죽지 않음, 계보에 영구 보존
```

**총 생애: 약 45~60일 (1.5~2개월)**

#### 진화 경로 (Good / Normal)

```
Good Care 경로 (포모도로 꾸준, 돌봄 잘 함):
  → 더 귀엽고 화사한 외형으로 진화
  → 진화 속도 빠름 (어덜트 18일)
  → 번식 성공률↑

Normal Care 경로 (포모도로 가끔, 돌봄 보통):
  → 일반적인 외형으로 진화
  → 진화 속도 표준 (어덜트 21일)
```

**Bad Care 없음:** 못생긴 캐릭터 대신 "느린 성장"으로 표현. 부정적 감정 제거.

#### Care Mistake 기준 (일일, Good Care에서 이탈)
```
+1 mistake: 배고픔 8시간+ 방치
+1 mistake: 행복도 0 유지 12시간+

0~1 mistake/일 → Good Care 유지
2+  mistake/일 → Normal Care로 전환
```

#### 절대 죽지 않는 방치 처리
```
방치 1일:   시무룩한 표정 (회복 가능)
방치 2~3일: "보고 싶어했어..." 메시지
방치 7일:   매우 슬픈 상태 (하트 1개)
방치 복귀:  먹이 + 포모도로 몇 회면 완전 회복
→ 언제든 돌아와도 캐릭터가 기다리고 있음
```

#### 은퇴 시스템
```
은퇴 = 끝이 아님

은퇴한 캐릭터:
  ├─ 컬렉션(명예의 전당)에 영구 보존
  ├─ 가끔 화면에 등장 "잘 지내니? 🥰"
  ├─ 가족 계보 트리에 영원히 기록
  └─ 은퇴 캐릭터 1마리당 포인트 획득 +5% 영구 버프
```

#### 가족 계보 트리
```
1세대: 🔥 불꽃이 (내가 키움, 45일, 은퇴)
         × 💧 물방울이 (친구가 키움)
         ↓
2세대: 🌫️ 증기 (함께 만든 새끼, 현재 키우는 중)
         ↓
3세대: 🥚 (알, 부화 대기)

표시:
  "불꽃이의 손자" "○○와 함께 만든 2세대"
  키운 일수, 달성 포모도로 수, Good/Normal 경로
```

### 4.3 돌봄 시스템

#### 상태 2가지 (심플하게)

```
배고픔 (Hunger): ❤️❤️❤️❤️ (0~4)
  감소: 6~8시간마다 -1 (하루 2~3회 먹이면 충분)
  회복: 음식 아이템 사용

행복도 (Happiness): ❤️❤️❤️❤️ (0~4)
  감소: 8시간마다 -1
  회복: 포모도로 완료 시 자동 +1
```

**에너지/수면/청결도 없음.** 돌봄 부담을 낮게 유지해 "소셜 집중"에 집중하도록.

#### 상태 감소 계산 방식 (정확도)
```
접속할 때마다 경과 시간 계산:

total_min = (지금 - 마지막_접속) + leftover_min
감소량 = floor(total_min / 360)     # 6시간 = 360분
leftover_min = total_min % 360      # 나머지 저장

→ 앱을 자주 열어도 정확하게 감소
→ leftover_min을 DB에 저장해 누락 방지
```

#### 음식 아이템

| 아이템 | 가격 | 배고픔 | 행복도 | 비고 |
|--------|------|--------|--------|------|
| 쌀밥 | 무료 (일 2회) | +2 | - | 기본식 |
| 스파게티 | 5pt | +3 | - | - |
| 초콜릿 | 7pt | +1 | +1 | 간식 |
| 딸기 케이크 | 10pt | +2 | +2 | 특식 |
| 프리미엄 밥상 | 20pt | +4 | +1 | 완전 회복 |

**무료 밥 일일 2회 한도:** 포인트 경제 균형 유지. 3회부터 포인트 필요.

#### 질병 (단순화, 1종류만)

```
아파요! 😷
  원인: 배고픔 0 상태 12시간 이상
  증상: 행복도 회복 안 됨
  치료: 만병통치약 (10pt)
  방치: 24시간 후에도 아픔 유지 (죽지는 않음)
```

### 4.4 포모도로

#### 타이머 설정
```
선택 가능: 15분 / 25분 / 50분
기본값: 25분
```

#### 완료 시 효과
```
포모도로 완료:
  ├─ 포인트 +5
  ├─ 행복도 +1 (자동)
  ├─ 오늘 집중 횟수 +1
  ├─ 공동 목표 카운터 +1
  └─ 캐릭터 "행복 점프" 애니메이션

약속 세션에서 완료 (보너스):
  ├─ 포인트 +5 추가 (총 +10)
  ├─ 친밀도 +1
  └─ "약속 지킴! 🎉" 메시지

타이머 스킵/중단:
  └─ 페널티 없음 (사용자 신뢰 우선)
```

#### 진화와의 연결
```
포모도로 누적이 진화 경로 결정

Good Care: 일일 포모도로 2회 이상 + care mistake 0~1
Normal Care: 일일 포모도로 1회 이하 OR care mistake 2+

→ "열심히 하면 캐릭터가 더 예쁘게 자란다"는 명확한 동기
```

### 4.5 함께 집중 (소셜 핵심)

**실시간(WebSocket) 없이 구현.** "동시 접속"이 아닌 "친구가 보고 있고, 친구와 비교된다"는 느낌을 준다.

#### A. 약속 세션 (Appointment)

```
흐름:
  1. "오늘 저녁 9시 같이 집중할래?" 제안 전송
  2. 친구 수락 → 둘 다 9시 알림
  3. 각자 포모도로 시작 (정확히 동시 아니어도 OK)
  4. 약속 시간대(9~10시) 내 둘 다 완료
  5. 보상: +10pt (각자), 친밀도+, "약속 지킴!" 배지

기술:
  - 시간대 겹침 체크만 (실시간 동기화 X)
  - 로컬 알림 (FCM 없이)

심리 효과:
  "약속했으니까 해야지" (사회적 책임감)
```

#### B. 친구 집중시간 경쟁 (Friend Focus Race)

```
목적:
  Forest의 함께 심기/전체 경쟁과 다르게 "가까운 친구끼리 누가 더 집중했는지"를 바로 보여준다.
  전체 유저 리더보드가 아니라 내 친구 관계 안의 작은 경쟁이다.

기본 지표:
  - 오늘 집중 시간: 완료된 포모도로 duration 합계
  - 이번 주 집중 시간: 월~일 완료된 포모도로 duration 합계
  - 완료 횟수는 보조 지표로만 표시

화면:
  친구 탭 상단에 "오늘의 집중 경쟁" 표시
  1위 친구 / 내 순위 / 친구와의 차이 표시
  예: "민지가 오늘 35분 앞서 있어요"
  예: "10분만 더 하면 1위예요"

보상:
  - 일일 1위: 작은 배지 + 포인트 +5
  - 주간 1위: 주간 왕관 배지 + 포인트 +20
  - 꼴찌 페널티 없음
  - 경쟁은 압박이 아니라 재시작 동기여야 함

기술:
  - 실시간 필요 없음
  - 포모도로 완료 시 기록 저장
  - 친구 랭킹은 DB에서 기간별 SUM(duration) 집계
  - 페이지 진입/완료 직후 새로고침으로 충분

MVP 우선순위:
  1단계: 오늘/이번 주 친구 랭킹 읽기 전용
  2단계: 일일/주간 배지
  3단계: 경쟁 결과가 캐릭터 친밀도/공동 알에 영향
```

#### C. 공동 목표 (Shared Goal)

```
설정:
  친구와 주간 목표 설정 ("둘이 합쳐 50회")

진행:
  나: 오전에 3회 + 오후에 2회 = 5회 기여
  친구: 저녁에 4회 기여
  합계: 완료 기록 기준으로 갱신되는 진행 바

달성 시:
  → 공동 알 획득 (번식의 씨앗)
  → +50pt 각자
  → "이번 주 함께 해냈어요!" 피드

기술:
  - 공동 카운터 집계 (DB only)
  - 실시간 통신 불필요
```

#### D. 솔로 모드 (친구 없을 때)

```
옵션 1: 혼자 포모도로 (기본 재미)
  → 캐릭터 양육 + 포인트 + 성장

옵션 2: 익명 매칭 카운터
  → "지금 1,234명이 함께 집중 중 🔥"
  → 나도 그 숫자의 일원
  → 심리적 동반 효과

→ 친구 없어도 완결성 있는 경험
→ 친구는 "보너스", 필수 아님
```

#### E. 친밀도 시스템

```
친밀도 획득:
  약속 세션 성공 → +2
  공동 목표 달성 → +3
  친구 경쟁에서 같은 날 둘 다 25분 이상 집중 → +1
  같은 날 둘 다 포모도로 → +1

친밀도 레벨 (0~10):
  Lv. 1~3: 캐릭터끼리 인사
  Lv. 4~6: 같이 놀기 가능 (행복도 +2)
  Lv. 7~9: 번식 신청 가능
  Lv. 10:  특별 이벤트 (초희귀 새끼 확률↑)
```

### 4.6 번식 & 세대교체 (Phase 2)

#### 번식 조건
```
1. 내 캐릭터: 어덜트 이상 (21일+)
2. 친구 캐릭터: 어덜트 이상
3. 친밀도: Lv. 7 이상
4. 하트 아이템: 3개 필요
```

#### 하트 아이템 획득
```
포모도로 10회 완료 → 하트 1개
주간 공동 목표 달성 → 하트 1개
연속 7일 포모도로 → 하트 2개

번식 1회 = 하트 3개
→ 약 30회 포모도로 = 번식 1회
→ 번식은 특별한 이벤트 (흔하지 않음)
```

#### 유전 & 외형 상속
```
부모 A × 부모 B → 자식

확률:
  40%: 부모 A 타입 (닮음)
  40%: 부모 B 타입 (닮음)
  15%: 혼합 타입 (희귀, 새로운 색상/특징)
  5%:  돌연변이 (완전 새로운 외형, 초희귀)

상속 요소:
  ├─ 기본 타입/색상
  ├─ 눈 모양 (부모 중 하나)
  ├─ 부모의 Good Care 경로라면 진화 시작점↑
  └─ 성격 (부모 평균)
```

#### 번식 흐름
```
1. 번식 제안 전송 (친구에게)
2. 친구 수락
3. 공동 알 생성 (72시간 부화)
4. 부화 가속: 포모도로 완료 시마다 진행도 +5%
   → 둘이 합쳐 포모도로를 많이 할수록 빨리 부화
5. 부화 완료 → 새끼 탄생 (양쪽 화면에 동시)
6. 새끼: 내 컬렉션에 추가 (최대 2~3마리 동시)
```

### 4.7 포인트 & 경제

#### 포인트 획득
```
포모도로 완료: +5pt
약속 세션 완료: +5pt 추가
공동 목표 달성: +50pt
일일 목표 달성 (4회): +10pt 보너스
연속 스트릭 (7일): +20pt
```

#### 포인트 사용
```
음식 아이템: 5~20pt
만병통치약: 10pt
꾸미기 (모자/리본): 15~30pt
하트 아이템: 50pt (번식 가속용)
배경/방 꾸미기: 20~40pt
```

#### 경제 균형
```
일일 평균 획득 (성실한 유저):
  포모도로 4회 → +20pt
  일일 목표 → +10pt
  합계: 약 30pt/일

일일 평균 소비:
  음식 (무료 초과분): ~5pt
  기타: ~5pt
  합계: 약 10pt/일

잔액 누적: ~20pt/일
→ 1주일: ~140pt (꾸미기 살 수 있음)
→ 인플레 방지: 꾸미기 가격을 충분히 높게 설정
```

---

### 4.8 게임 중독 방지

#### 기본 원칙
```
포모도로가 항상 선행되어야 한다.
게임 요소는 집중의 "결과"이지 "목적"이 아니다.
```

#### 구체적 장치
```
1. 행복도 회복 = 포모도로 완료 자동 (버튼 없음)
   → 게임만 하면 행복도 못 올림

2. 번식 재료 (하트) = 포모도로로만 획득
   → 번식하고 싶으면 집중해야 함

3. 진화 경로 = 포모도로 꾸준함이 결정
   → 예쁜 캐릭터 원하면 집중해야 함

4. 꾸미기 아이템 = 포인트 (포모도로로만 획득)
   → 꾸미고 싶으면 집중해야 함
```

## 5. 화면 설계

### 탭 구조 (하단 4개)

```
🏠 홈       → 캐릭터 + 포모도로 타이머
👥 친구     → 집중시간 경쟁 + 약속 세션 + 공동 목표 + 친밀도
⭐ 캐릭터   → 성장 단계 + 진화 + 가족 계보
🛍️ 스토어   → 아이템 구매
```

### 화면별 상세

#### 홈 화면
```
[상단] 포인트 | 오늘 집중 시간 | 오늘 집중 횟수
[캐릭터] 화면 60% 차지, 표정 자동 변화
[상태바] 배고픔 ❤️❤️❤️ | 행복도 ❤️❤️❤️❤️
[타이머] 시간 선택 (15/25/50) → 원형 타이머
[버튼] 포모도로 시작 / 먹이기 / 청소
```

#### 친구 화면
```
[집중 경쟁] 오늘/이번 주 친구 순위 + 내 순위 + 1위와의 차이
[친구 카드] 캐릭터 + 이름 + 오늘 집중 시간
[약속 세션] 시간 선택 → 제안 → 수락 → 알림
[공동 목표] 진행 바 (합산) → 달성 → 공동 알
[친밀도] 두 캐릭터 나란히 + 레벨 표시
```

#### 캐릭터 화면
```
[기본 정보] 이름 / 타입 / 나이 / 단계
[성장 진행] 단계 시각화 (알~은퇴)
[진화 경로] Good / Normal (현재 경로 표시)
[가족 계보] 트리 구조 (잠금 → 번식 후 열림)
[컬렉션] 은퇴한 캐릭터들
```

#### 스토어 화면
```
[포인트 잔액 표시]
탭: 음식 / 꾸미기 / 특별
각 아이템: 이미지 + 효과 설명 + 가격 + 구매 버튼
구매 = 즉시 사용 (인벤토리 없음)
```

## 7. 개발 로드맵

### 단계별 개발 원칙

```
한 번에 MVP 전체를 만들지 않는다.
각 단계는 "혼자 써볼 수 있음" 또는 "친구 1명과 써볼 수 있음" 상태로 끝난다.
앞 단계의 사용성이 확인되기 전에는 다음 단계의 복잡한 기능을 만들지 않는다.
```

### Step 0 - 클릭 가능한 뼈대

```
목표: 앱의 첫인상과 흐름 검증
- Next.js + Tailwind CSS 셋업
- 홈 / 친구 / 캐릭터 / 스토어 탭 껍데기
- 목업 캐릭터 1종
- 목업 포모도로 화면
- 실제 DB/인증 없음

완료 기준:
- 사용자가 첫 화면에서 "무엇을 하는 앱인지" 10초 안에 이해한다
```

### Step 1 - 솔로 포모도로

```
목표: 가장 쉬운 핵심 행동부터 구현
- 15/25/50분 포모도로 타이머
- 완료 기록 저장 (초기에는 localStorage 가능)
- 오늘 집중 시간 / 오늘 완료 횟수 표시
- 완료 시 캐릭터 행복 애니메이션 1개
- 포인트 +5 지급

제외:
- 회원가입
- 친구
- 돌봄 상태 감소
- 스토어

완료 기준:
- 개발자 본인이 하루에 실제 포모도로 3회 이상 기록할 수 있다
```

### Step 2 - 계정과 서버 저장

```
목표: 기록이 사라지지 않는 개인 앱으로 만들기
- 회원가입 / 로그인 / JWT
- PostgreSQL에 포모도로 기록 저장
- 오늘/이번 주 집중 시간 집계
- 기본 캐릭터 1마리 저장
- 포인트 누적 저장

완료 기준:
- 다른 기기/브라우저에서 로그인해도 집중 기록이 보인다
```

### Step 3 - 친구와 집중시간 경쟁

```
목표: 가장 가벼운 소셜 재미 먼저 구현
- 친구 초대 링크/코드
- 친구 목록
- 오늘 친구 집중시간 순위
- 이번 주 친구 집중시간 순위
- "친구보다 15분 뒤처짐" 같은 차이 메시지
- 일일 1위 배지 또는 +5pt 보상

제외:
- 약속 세션
- 공동 목표
- 공동 알
- 실시간 WebSocket

완료 기준:
- 친구 2~3명이 서로의 집중 시간을 보고 추가 포모도로를 시작한다
```

### Step 4 - 최소 돌봄 시스템

```
목표: 캐릭터가 단순 장식이 아니라 기록의 보상이 되게 만들기
- 배고픔 / 행복도 2개 상태
- 포모도로 완료 시 행복도 +1
- 무료 밥 일 2회
- 먹이기
- 시무룩/행복 표정

제외:
- 질병
- 복잡한 진화
- 꾸미기 스토어

완료 기준:
- 사용자가 "집중하면 캐릭터가 좋아진다"를 이해한다
```

### Step 5 - 약속 세션

```
목표: 친구 경쟁 다음 단계의 책임감 추가
- "오늘 9시에 같이 집중할래?" 제안
- 친구 수락
- 약속 시간대 내 완료 여부 체크
- 둘 다 완료 시 보너스 +5pt
- 친밀도 +2

알림:
- 초기에는 앱 안 알림/표시로 시작
- 브라우저 푸시/FCM은 검증 후 별도 단계

완료 기준:
- 친구 1명과 약속을 잡고 둘 다 완료 처리할 수 있다
```

### Step 6 - 공동 목표와 공동 알

```
목표: 장기 차별점의 가장 작은 버전 구현
- 주간 공동 목표 생성
- 둘의 완료 횟수 또는 집중 시간 합산
- 목표 달성 시 공동 알 획득
- 공동 알은 처음에는 "잠금 해제된 보상" 수준으로 단순 표시

제외:
- 실제 유전 시스템
- 혼합 캐릭터 생성
- 가족 계보 UI

완료 기준:
- 사용자가 "친구와 함께 만든 보상"을 처음 경험한다
```

### Step 7 - 베타 MVP

```
목표: 10~20명에게 배포 가능한 첫 버전
- 온보딩
- 기본 오류 처리
- 통합 테스트
- 포인트 밸런싱 1차
- 모바일 화면 정리
- Vercel + Render 배포

완료 기준:
- 10~20명 베타에서 D1/D7, 친구 연결, 경쟁 참여 데이터를 수집한다
```

### Phase 2 - 번식 & 계보

```
- 공동 알 부화
- 공동 번식 (단순 유전 시스템)
- 혼합 캐릭터
- 가족 계보 트리
- 은퇴 시스템 + 명예의 전당
- 하트 아이템 획득 흐름
```

### Phase 3 - 확장

```
- 진짜 실시간 함께 집중 (WebSocket)
- 유령 세션 (친구 과거 기록으로 함께하는 느낌)
- 그룹 집중 (3명+)
- 전체/시즌 리더보드
- Premium 구독 출시
```

## 9. 성공 지표 (KPI)

```

가설 2: "함께 집중하면 혼자보다 완료율이 높다"
측정: 약속 세션 완료율 vs 솔로 포모도로 완료율

```

## 부록: 주요 의사결정 이력

| 결정 | 이유 |
|------|------|
| 단계별 MVP | 처음부터 모든 기능을 만들면 검증 전에 범위가 커짐 |
| 친구 집중시간 경쟁을 초기 소셜로 배치 | 약속/공동목표보다 구현이 쉽고 Forest와 다른 친구 단위 재미를 빠르게 검증 가능 |
| 실시간 X → 약속+공동목표 | WebSocket 복잡도, "함께 느낌"은 비동기로 충분 |
| 캐릭터 불사 | 죽으면 이탈율 높음, 애착 보호 |
| 생애 45~60일 | 정들기 위한 최소 시간 |
| 상태 2종만 | 소셜이 주인공, 돌봄 부담 낮게 |
| Bad Care 없음 | 부정적 경험 제거, "느린 성장"으로 대체 |
| 리롤 3회 | 랜덤인데 원하는 캐릭터 못 받는 불만 해결 |
| 공동 번식은 Phase 2 핵심 | "함께 만든 캐릭터"는 강한 차별점이지만 초기 구현 난이도가 높음 |
| 솔로 모드 | 양면 시장 문제 (친구 없어도 완결) |
| 꾸미기는 베타 이후 | 포인트 사용처는 필요하지만 초기 핵심 검증에는 불필요 |
| 온보딩 1분 체험 | 25분 기다리면 이탈 |
